## Step 4: Critical Rules

### NEVER Modify:
- Any .js files in /jsGasoum/ folder
- Any backend server files
- Any <script> tags or JavaScript function calls
- The id attributes on elements (JavaScript depends on these)
- The onclick, onchange, or other event handlers
- Form field names or IDs
- DataTable button configurations (keep as simple array: ["copy", "excel", "pdf", "print"])
- Variable declarations like `tblDT = ` (don't add `var` if original didn't have it)

### ONLY Modify:
- HTML structure and layout
- CSS class names (update to new theme classes)
- CSS files (if creating page-specific styles)
- Remove old template elements (header, right panel, etc.)

### NO COMMENTS RULE (CRITICAL):

NEVER add comments to any file:

WRONG - Never do this:
<!-- This is the header section -->
<div class="br-logo">...</div>

/* Button styles */
.btn-primary {
  background: var(--accent);
}

CORRECT - Clean code without comments:
<div class="br-logo">...</div>

.btn-primary {
  background: var(--accent);
}

Rules:
- Do NOT add HTML comments (<!-- -->)
- Do NOT add CSS comments (/* */)
- Remove any existing comments you encounter while converting
- Keep code clean and self-documenting through class names
- The only exception: Keep existing comments in JavaScript files (don't touch JS files anyway)

### CSS & Styling Rules (CRITICAL):

NEVER use inline styles:

WRONG - Never do this:
<div style="margin-top: 20px; color: red;">...</div>
<button style="background: blue;">...</button>

CORRECT - Always use classes:
<div class="custom-section">...</div>
<button class="btn btn-primary">...</button>

When you need custom styling:

1. First, check if existing CSS classes can solve it:
   - Review base.css for utility classes
   - Review components.css for component styles
   - Review people.css for table/button styling
   - Review page-specific CSS files (e.g., cus.css, cash.css)

2. If existing classes don't work, create NEW page-specific CSS:
   
   Example: Converting stock_main.html
   
   → Create: /lecs_dev2.0/cssGasoum/stock.css
   → Add styles there
   → Link it in stock_main.html: <link rel="stylesheet" href="../cssGasoum/stock.css?v=1">

3. If the style is reusable across multiple pages:
   - ASK FIRST: "This styling might be needed on other pages. Should I add it to components.css or create a new file?"
   - Wait for user confirmation before modifying shared CSS files

4. File naming convention for new CSS:
   
   Page-specific: [page-name].css
   Examples:
   - stock.css (for stock-related pages)
   - cash.css (for cash-related pages)  
   - active_tabs.css (for appointment pages)

5. When modifying existing CSS files:
   - ALWAYS ASK FIRST if changes might affect other pages
   - Only modify if you're 100% certain it won't break anything
   - Better to create a new file than risk breaking converted pages

Examples:

WRONG - Inline style:
<div class="content-card" style="max-width: 900px; padding: 30px;">
  ...
</div>

CORRECT - Create new CSS file:

In stock.css:
.stock-card {
  max-width: 900px;
  padding: 30px;
}

In HTML:
<div class="content-card stock-card">
  ...
</div>

WRONG - Inline style for one-off need:
<button style="margin-right: 10px; background: #f6d32d;">Save</button>

CORRECT - Use existing utility class or create specific class:

Option 1: Use existing utility:
<button class="btn btn-primary mg-r-5">Save</button>

Option 2: Create new class in page CSS:

In page-specific.css:
.btn-stock-action {
  margin-right: 10px;
  background: var(--accent);
}

In HTML:
<button class="btn btn-stock-action">Save</button>

CSS Decision Tree:

Need styling? →
  ├─ Does a class exist in base.css/components.css/people.css? → Use it ✅
  ├─ Is it page-specific? → Create [page].css ✅
  ├─ Is it reusable across multiple pages? → ASK USER FIRST ⚠️
  └─ Should I use inline style? → NEVER ❌

### ALWAYS ASK Before:
- Making any decision you're uncertain about
- If you encounter unusual HTML structure that doesn't match the pattern
- If content seems incomplete or you need to reference the old design
- If you need clarification on which elements to keep/remove
- Before creating any new CSS files
- Before modifying existing shared CSS files (base.css, components.css)
- If you're unsure whether styling should be in a new file or existing file
- If you find yourself wanting to use inline styles (there's always a better way!)
- If you're tempted to add comments (don't - just ask if structure is unclear)

### Stats Pages - Calendar Showing at Bottom

On stats pages with datepickers, the jQuery UI calendar may appear at the bottom of the page. Add this line after the datepicker initialization:

```javascript
$(".fc-datepicker").datepicker({
  showOtherMonths: true,
  selectOtherMonths: true,
  dateFormat: "dd/mm/yy",
});
$("#ui-datepicker-div").hide();  // ADD THIS LINE
initializeDates();
```

### SMS Campaign Form Alignment

For complex forms with inline filters (like sms_campaign.html), use the pattern in sms.css:
- `.sms-form` container with max-width
- `.sms-filter-row` for flex alignment of checkboxes, labels, and inputs
- Consistent gaps and minimum widths for form elements
- Avoid `form-inline` - use Bootstrap grid + custom flex classes instead

## Step 9: JavaScript Pattern for New Pages

Replace the old JavaScript initialization:

OLD (remove these):
```javascript
menu_constructor();
information_system_constructor(false, 0);
menu_user_info_constructor();
menu_notifications();
showUserName();
footer_constructor();
```

NEW (use this):
```javascript
credsAppOtherCheck();
if (typeof menu_constructor === "function") menu_constructor();

document.addEventListener("click", function (e) {
  var link = e.target.closest(".br-menu-link");
  if (!link) return;
  var sub = link.nextElementSibling;
  if (sub && sub.classList && sub.classList.contains("br-menu-sub")) {
    e.preventDefault();
    link.classList.toggle("show-sub");
    return;
  }
});

(function () {
  try {
    var raw = localStorage.getItem("lcObj");
    if (!raw) return;
    var u = JSON.parse(raw);
    var name = (u && (u.name || u.username) ? String(u.name || u.username) : "").trim();
    var surname = (u && u.surname ? String(u.surname) : "").trim();
    var full = (name + (surname ? " " + surname : "")).trim();
    if (!full) return;
    var el = document.getElementById("lc_user_name");
    if (el) el.textContent = full;
  } catch (e) {}
})();

try {
  document.getElementById("lcMenuId").classList.add("active");
  document.getElementById("lcMenuId").classList.add("show-sub");
  document.getElementById("lcSubmenuId").classList.add("active");
} catch (e) {}
```
